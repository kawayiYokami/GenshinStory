# UI Protocol Layer

这个目录用于存放“前端内置交互协议”，不属于外部 `tools` 调用。

典型场景：
- `ask_choice`：向用户发起选择题交互
- `emoji_reaction`：展示表情反馈
- 其他仅由前端消费、无需后端执行的 UI 指令

## 设计原则

- 与 `tools/` 严格隔离：这里不是 MCP/函数调用工具。
- 面向 UI：协议只描述前端该如何展示和交互。
- 可扩展：后续可新增更多 UI 指令类型。

## 当前建议

- 解析层：模型输出先进入统一解析，再分发到 `tools` 或 `ui-protocol`。
- 执行层：`ui-protocol` 只做渲染/状态更新，不做外部 API 执行。
